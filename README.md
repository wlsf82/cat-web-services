# CWS: Cat Web Services

A satirical fake cloud console where cats manage their humans like underperforming infrastructure. Inspired by AWS-style dashboards, CWS lets you monitor human compliance metrics, trigger escalations, and file incidents — all from a polished, cat-operated control plane.

## What it does

The app is a single-page console at `/` built around six parody cloud services. Cats log in with a breed-verified account and interact with a live household dashboard that updates in real time.

### Services

| Key | Short name | Category | What it tracks |
| --- | --- | --- | --- |
| `catops` | CatOps | Operations | Human uptime, petting availability, lap readiness |
| `iam` | IAM | Security | Permission control — petting, feeding, sofa access |
| `clawedwatch` | ClawedWatch | Observability | Real-time alarms for vacuum exposure and snack withholding |
| `sns` | SNS | Messaging | Snack Notification Service — treat-request escalations |
| `s3` | S3 | Storage | Scratch, Sleep, Store — nap and box inventory |
| `route9` | Route 9 | Networking | Route 9 Lives — room traversal and zoomie routing |

Each service has a health status (`healthy` / `degraded` / `chaotic`), a sparkline chart, and a detail panel.

### Human Status Card

Tracks your primary human across five live metrics that change when you fire actions:

- **Lap readiness** — probability the human will offer a suitable lap
- **Treat latency** — seconds until snack delivery
- **Meow acknowledgement** — response rate to vocal complaints
- **Blanket warmth** — thermal comfort score
- **Vacuum threat** — `low` / `elevated` / `critical`

The human's overall compliance state (`compliant`, `available`, `distracted`, `sleeping`, `unreliable`) is derived from the average of the three primary metrics.

### Action Center

Five interactive actions mutate the human status in real time and append an incident to the feed:

| Action | Effect |
| --- | --- |
| **Deploy Meow** | Raises meow acknowledgement, reduces treat latency |
| **Wake Human** | Increases lap readiness and blanket warmth |
| **Request Treat** | Reduces treat latency, increments snack request count |
| **Trigger Guilt Stare** | Raises lap readiness and meow acknowledgement |
| **Emergency Zoomies** | Reduces blanket warmth, increases treat latency |

### Incident Feed

A live log of up to seven incidents generated by actions, each tagged with a severity level (`info`, `warning`, `critical`) and a timestamp.

### ClawedShell

A resizable terminal dock anchored to the bottom of the page. Drag the handle to resize or double-click to expand/collapse. Supported commands:

```text
help      list available commands
status    show the currently focused service and household health
recent    list recently viewed services
whoami    print the logged-in cat account details
purrr     purr
clear     reset the terminal output
```

### Cat Account System

Visitors can register a viewer account by picking a display name and selecting a cat breed. Registration fetches the breed list and a breed avatar from [The Cat API](https://thecatapi.com). Accounts are persisted in `localStorage` — no backend required. Multiple accounts can be saved and switched between via the sign-in flow.

### Service Search

The top bar includes a live search that filters the service grid by name, short name, description, or category.

## Stack

| Layer | Technology |
| --- | --- |
| UI framework | React 18 + TypeScript |
| Build tool | Vite 6 |
| Routing | React Router v7 |
| Animation | Framer Motion |
| Icons | Lucide React |
| Styles | SCSS Modules |
| External API | [The Cat API](https://thecatapi.com) (breeds + images) |
| Deployment | Vercel (SPA rewrite config included) |

## Local setup

```bash
npm install
npm run dev
```

## Build

```bash
npm run build       # type-check + production bundle
npm run preview     # serve the production build locally
```

## Project structure

```text
src/
  components/       # Shared layout and UI primitives
  config/           # Static cat breed and service logo config
  data/             # Mock service definitions, incidents, and actions
  features/         # Feature-level components (actions, auth, terminal, etc.)
  pages/            # Route-level page components
  types/            # Shared TypeScript types
  utils/            # Business logic: catApi, cwsEngine, viewerAccount
```
